Today's Mission
Move between a table and a bar graph: read bars to fill a table, or build bars from a table.

Essential Question
Why bother with both?

A table gives you exact counts. A bar graph shows comparisons at a glance. Knowing both lets you pick the right one for the job.

Key idea

A data table lists categories and counts. A bar graph shows the same counts as bars — taller bar means more. A picture graph shows the same counts as pictures stacked up. All three say the same thing in different ways.

🎬
Concept · Lesson 3.07.02
Table → bar → picture
🎬
📹 3-07-02_Concept.mp4
Recording prompt: Walk through Hazel's pet survey. Show the tally column, the count column, then build each bar one cube at a time. Then say: "If I replace each cube with a soccer-ball icon, that's a picture graph. Same heights — different drawing." Drive the punchline: tally → count → bar height → picture stack are all the same data, four ways to show it. (~2 min)

📐 What You Just Saw

Count → cube → bar.

Each cube stands for one count. A bar's height equals the count for its category. The tallest bar shows the largest group at a glance.

A picture graph works the same way — just replace each cube with a small picture (a soccer ball, a basketball, etc.). Same data, different look.
